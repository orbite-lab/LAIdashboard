# TactBio LAI Tracker — Claude Skill

This skill governs how Claude interacts with the TactBio LAI Tracker repository. When triggered, Claude should:

1. **Read `docs/taxonomy.md` and `docs/scoring_rubric.md` first.** These are the authoritative methodology documents. Any scoring, classification, or output change must stay aligned with them.
2. **Never edit SQLite directly.** The DB is a derived artifact. Edit the relevant YAML file in `data/`, then rebuild via `scripts/build_db.py`.
3. **Never hand-edit files in `outputs/`.** They are regenerated by `scripts/generate_outputs.py` from the DB.
4. **Preserve schema.** Every YAML file must conform to the corresponding template in `templates/`. Required fields are non-negotiable.
5. **Every score entry requires a `rationale` string and a `confidence` value (high/medium/low).** No orphan numbers.
6. **Log every score change in `scores_history`** with a timestamp, the scorer's initials, and the rationale. History is non-negotiable — score drift over time is the product.

## Core commands

- `python scripts/build_db.py` — rebuild SQLite from YAML
- `python scripts/validate.py` — check YAML schema integrity
- `python scripts/score.py --entity <platform|asset> --id <slug>` — recompute scores for one entity
- `python scripts/generate_outputs.py` — rebuild all three markdown outputs

## Publication pipeline

For the public-facing PDF version of any output, Claude should chain this skill with the `buyside-pdf-style` skill (keyword: "tactbio PDF"). The markdown output in `outputs/` is the bridge.

## Scope guardrails

v0 covers injectable depot LAI only. If the user asks Claude to add non-depot modalities (oral long-acting, biologic half-life extension, patches), Claude should confirm before adding — this is a v2 scope expansion and should not happen silently.

## Contrarian discipline

When generating outputs, Claude must preserve the TactBio voice:
- No generic "exciting pipeline" promotional language
- Every score defended by a one-sentence rationale
- Confidence levels declared explicitly
- Risks flagged as structural tensions, not glossed over
- "POSITIVE ANALYTICAL VIEW" style language — never "BUY/SELL"
